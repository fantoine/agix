## `remove` / `update` — mangled lock source aborts uninstall (Task 4 review)

**Scenario:** Pre-existing lock entry whose `source` string doesn't parse via `SourceSpec::parse`.
**Command:** `agix remove <pkg>` or `agix update`
**Expected:** warn and fall back to file-based uninstall so user can still clean up.
**Actual:** `Installer::uninstall` in `src/core/installer.rs:184` does `SourceSpec::parse(&pkg.source)?` — any parse error aborts before any files are removed or the lock entry is cleaned.
**Severity:** minor (would only bite on hand-edited or older-format lockfiles).
**Fix commit:** Deferred to Task 15 (review `remove`).

## `add marketplace` — total failure returns Ok (Task 4 review)

**Scenario:** `agix add marketplace <bad-ref>` where every target CLI fails.
**Command:** `agix add marketplace <bad/ref>@<plugin>`
**Expected:** non-zero exit (CI pipelines should detect failure).
**Actual:** `installer.rs:35-97` turns each per-driver failure into `output::warn` and continues; `install_manifest` returns `Ok(())` even if no driver succeeded. Contrast with non-marketplace branch which propagates via `?`.
**Severity:** minor (affects CI ergonomics only; warnings still visible).
**Fix commit:** Deferred to Task 10 (review `add`).

## `install` — `claude plugin install` on already-installed plugin warns non-zero (Task 4 review)

**Scenario:** Re-run `agix install` on a manifest with a marketplace plugin already installed via `claude`.
**Command:** `agix install`
**Expected:** silent success (idempotent).
**Actual:** `claude plugin install <x>@<y>` exits non-zero when already installed; `ClaudeCodeDriver::install_marketplace_plugin` bubbles this as `AgixError::Other`, which the installer surfaces as `warn`.
**Severity:** cosmetic (correct state is reached; noise only).
**Fix commit:** Deferred to Task 12 (review `install`).

## `doctor` — marketplace packages silently pass (Task 4 review)

**Scenario:** Marketplace-installed plugin files deleted manually from `~/.claude/plugins/`.
**Command:** `agix doctor`
**Expected:** warn that the plugin is missing or at least note `marketplace (managed by <cli>)`.
**Actual:** `src/commands/doctor.rs:24-30` iterates `pkg.files`, which is empty for marketplace packages — loop is a no-op, doctor reports "All installed files present".
**Severity:** minor (misleading but not incorrect — we don't own those files).
**Fix commit:** Deferred to Task 13 (review `doctor`, Phase B).
